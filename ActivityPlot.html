Analyse de l'activité des paquets et dépôts R¶

Ce notebook a pour but de fournir quelques étapes (et le code associé) permettant de visualiser l'activité des dépôts et des paquets R. Dans un premier temps, nous chargeons les noms et propriétaires des dépôts R qui correspondent à un package R (autrement dit : qui ont un fichier DESCRIPTION). Ensuite, nous chargeons la liste des événéments du flux d'activités de chacun de ces dépôts, et nous comparons par groupe : les dépôts qui ne sont pas des packages, les dépôts qui sont des packages, les dépôts qui sont des packages SAUF ceux de cran et rpkg (deux owner particuliers).

Chargement du flux d'activités¶

Nous allons extraire les informations du flux d'activités. En particulier, nous garderons : - Le nom et le propriétaire du dépôt, - Le type de l'événement, - La date de l'événement

In [101]: 
import pymongo
import dateutil.parser
from itertools import imap

conn = pymongo.MongoClient()

def _from_json_to_flat(e):
    return {'type': e['type'], 
                   'created_at': dateutil.parser.parse(e['created_at']),
                   'owner': e['repository']['owner'],
                   'repository': e['repository']['name']}

events = imap(_from_json_to_flat, 
              conn.r.events.find({'type': 'PushEvent'},
                                 fields=['type', 'created_at', 'repository.name', 'repository.owner']))
In [102]: 
# This should take some time (expect 683425 items)
df_events = pandas.DataFrame(list(events))

Graphiquement...¶

Le graphique suivant reprend une moyenne glissante sur 30 jours du nombre d'événements de type "PushEvent". Chaque courbe correspond à une partie (ou le tout) des dépôts : - la courbe bleue correspond aux dépôts contenant un package R lié au compte cran, - la courbe verte est invisible, mais elle correspond à un tout petit trait aux alentours d'avril 2014 (un peu avant le début de la courbe bleue) et correspond aux dépôts contenant un package R lié à rpkg, - la courbe rouge correspond aux dépôts contenant un package R, qui n'appartient ni à rpkg, ni à cran. - la courbe cyan correspond à l'ensemble de tous les dépôts liés à R.

Activité des dépôts contenant un package sur CRAN et en dehors¶

Nous allons utiliser les informations collectées depuis CRAN afin d'identifier quels sont les packages R disponibles sur Github qui sont également présents sur CRAN. Cela nous permettra de comparer l'activité des dépôts qui sont liés à un package "officiel" de ceux qui ne le sont pas.

Pour procéder à cette identification, nous comparons le champ Package des fichiers DESCRIPTION des packages R de Github avec ceux de CRAN.
