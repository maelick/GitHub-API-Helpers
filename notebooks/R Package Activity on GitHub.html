Activité des paquets R sur GitHub¶

Chargement des données¶

Nous avons collect pas mal de donnes propos des paquets R prsents sur GitHub. En particulier, nous avons le flux d'activit de tous ces paquets (avec notamment les PushEvents) ainsi que des fichiers .csv contenant les fichiers DESCRIPTION de chaque package. Ces fichiers ont t collects la fois sur CRAN, sur BioConductor et sur GitHub.

Il serait intressant de comparer l'activit de ces diffrents groupes de paquets.

Pour ce notebook, les fichiers suivants vont tre utiliss : - bioconductor_description.csv : contient les informations de DESCRIPTION des packages prsents sur BioConductor. - cran-description.csv : idem mais pour CRAN. - rforge_description.csv : idem mais pour RForge. - github-description.csv : idem mais pour tous les packages sur GitHub. - github_R_pushevents.csv : contient les vnements de type PushEvent relatifs aux dpts contenant un package R sur GitHub.

Mise en place des DataFrames¶

Ceux qui ncessitent juste un renommage sur le label des packages.

cran doit tre pivot et ncessite galement un renommage et un index sur le nom des packages. Le dataframe cran contient galement des informations pour plusieurs versions d'un mme package. Par convention, nous ne garderons que la dernire information (i.e. la dernire version).

In [67]: 
cran = cran.rename(columns={'package': 'Package'}).drop(labels=['version'], axis=1).drop_duplicates(['Package', 'key'], take_last=True).pivot(index='Package', columns='key', values='value')

Le dataframe github est un peu diffrent, dans le sens o l'identification se fait via le couple (owner, repository) et non pas via le nom du package. La premire chose faire est d'associer un nom de package chaque dpt. Ensuite, nous utiliserons (owner, repository) comme index.

In [68]: 
_ = github[github['key'] == 'Package'][['owner', 'repository', 'value']].rename(columns={'value': 'Package'})
_ = github.merge(_, on=('owner', 'repository')).drop_duplicates(['owner', 'repository', 'key'], take_last=True)
_['url'] = _['owner'] + '/' + _['repository']
github = _.pivot(index='url', columns='key', values='value')

Enfin, le dataframe events sera utilis ultrieurement afin de joindre les informations des autres dataframes. La colonne package va tre renomme afin de faciliter la jointure avec bioconductor et cran, alors qu'une nouvelle colonne url va tre construite afin de faciliter la jointure avec github. Finalement, ce dataframe va tre index sur base de la date de l'vnement, renomme en date.

Ces deux owners servent en ralit mirrorer les paquets qui sont prsents sur CRAN, et ne reprsentent pas un dveloppement de paquet ou quoi que ce soit rellement li au dveloppement de paquets. Identification de l'appartenance de chaque événement à une ou plusieurs communautés¶

Chaque vnement est associ un dpt GitHub ainsi qu' un package.

Une remarque importante : l'association entre un vnement et un package provient d'un pr-traitement en amont, dont le rsultat a t stock dans le .csv la base de events. Si, lors d'une reproduction de l'exprience courante, l'utilisateur souhaite repartir des donnes brutes (issues de MongoDB ou de GitHubArchive), il convient de faire une jointure avec le dataframe github, sur base de url afin d'associer un package au dpt concern par l'vnement.

Grce cette association, il nous est possible d'identifier quels sont les packages qui appartiennent telle ou telle communaut (ex. : CRAN, BioConductor ou GitHub exclusivement) et de mesurer les diffrences d'activit en fonction de ces communauts. La premire tape consiste donc identifier, pour chaque vnement, si le dpt relatif est li une (ou plusieurs) communauts.
