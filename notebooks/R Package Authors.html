Identification des auteurs de packages¶

Nous allons rcuprer, pour chaque communaut, la liste des auteurs de packages. Nous essayerons ensuite de faire un merging des identits, et d'identifier la taille de chaque communaut en terme d'auteurs, ainsi que les auteurs qui sont prsents dans plusieurs communauts.

Nous allons devoir parcourir les champs Author et rcuprer la liste des auteurs. Ensuite, il conviendra de "nettoyer" chaque auteur pour obtenir une sorte d'identit canonique qui servira de base pour le matching. L'approche est discutable, mais a le mrite d'tre relativement simple. Nous pourrons manuellement attester de la qualit de cette approche.

La fonction split_author_list se base sur l'expression rgulire suivant pour "dcomposer" la liste des auteurs (format textuel) en une liste d'auteurs. Les sparateurs sont lists dans RE_SPLIT_AUTHORS.

In [4]: 
import re

RE_SPLIT_AUTHORS = re.compile(r'(,|;|( \- )|  | & |(and)|(with)|(contributions from))')

def split_author_list(authors):
    candidates = re.split(RE_SPLIT_AUTHORS, authors)
    return map(lambda x: x.strip(), filter(lambda x: x is not None and len(x) > 5, candidates))

La fonction clean_name utilise l'expression RE_GREP_AUTHOR pour identifier les auteurs. Grosso-modo, on procde comme suit :

  • On dcoupe la chane correspondant potentiellement un auteur avec le sparateur ' ' et '\n'.
  • Chaque morceau est pass en minuscules.
  • Au sein de chaque morceau, on rcupre la plus longue suite conscutive de caractres autoriss ('a' 'z', plus '-') d'au moins 3 caractres, en dbut de chane et se terminant soit en fin de chane, soit par un '.'. Cela permet d'liminer les initiales, tout en conservant les listes d'auteurs spars par un '.'.
  • Ces diffrents morceaux ainsi rcuprs sont tris par ordre alphabtique (pour obtenir une sorte de version "canonique").
In [5]: 
RE_GREP_AUTHOR = re.compile(r'^([a-z\-]{3,}).?$')
def clean_name(name):
    name = name.replace('\n', ' ')
    _ = name.lower().split(' ')
    names = []
    for e in _:
        for f in re.findall(RE_GREP_AUTHOR, e):
            names.append(f)
    names.sort()
    return names

Le bloc suivant va composer le dictionnaire name. Ce dictionnaire contiendra comme cls les noms canoniques des auteurs. Chaque auteur est associ un dictionnaire contenant, pour chaque source, l'ensemble des packages pour lesquels cet auteur est repris. Une entre supplmentaire names contient galement la liste des noms qui ont t associs cet auteur (pour vrification manuelle, par exemple).

names = {}
for source_name, source in sources:
    for index, value in source.fillna('').iterrows():
        for dirty in split_author_list(value['Author']):
            author = ' '.join(clean_name(dirty))
            if len(author) > 0:
                d_author = names.setdefault(author, {})
                d_author.setdefault('names', set()).add(dirty)
                d_author.setdefault(source_name, set()).add(index)

La fonction suivante (un peu moche, et qui ncessite des variables globales) va retourner un dictionnaire avec comme cls chaque source (CRAN, GitHub, etc.). Comme valeur associ cette cl, un ensemble des auteurs prsents dans cette source. Le paramtre N permet de dfinir un "nombre minimum de packages distincts dvelopper pour tre considr comme auteur".

def _author_sets(N = 1):
    authors = {source[0]: set() for source in sources}
    for name, dname in names.iteritems():
        pkg = set()
        for k,p in dname.iteritems():
            if k != 'names':
                pkg = pkg.union(p)
        if len(pkg) >= N:
            for key in dname:
                if key != 'names':
                    authors[key].add(name)
    return authors
